Git rebase digunakan untuk mengintegrasikan perubahan dari suatu branch ke branch yang lain. Ketika bekerja di suatu branch dan anggota tim lain mengupdate branch utama dengan commit baru, akan muncul forked history. Perintah git rebase akan memindahkan feature branch ke ujung dari main branch dengan merewrite project history dan membuat commit baru untuk setiap commit dari branch semula. Git rebase memberikan project history yang lebih clean jika dibandingkan dengan perintah git merge.
